# Glance 랜딩 페이지 — 새 세션 브리핑

## 이 문서의 목적
Glance 랜딩 페이지를 새 세션에서 만들기 위한 컨텍스트 패키지.
이 파일 하나로 기존 세션의 모든 제품 결정을 이어받을 수 있다.

---

## 제품 개요

**Glance**는 채널톡 같은 CS 플랫폼에 붙이는 AI 컨텍스트 레이어다.
챗봇이 해결 못한 고객이 매니저에게 전환되는 순간,
매니저가 대화를 처음부터 읽지 않아도 되게 해주는 우측 패널 도구.

### 한 줄 포지셔닝
> "고객 전환 순간, 10초 읽기를 0초로"

### 타깃 유저 (페르소나: "지수씨")
- 소규모 이커머스 운영자 또는 CS 전담 담당자
- 1~5인 팀, 채널톡으로 CS 운영 중
- 하루 수십 건 상담, ALF(챗봇) 에스컬레이션 케이스가 반복됨
- 핵심 불편: 챗봇 대화를 처음부터 읽고 + 감정 파악 없이 응대 시작

### 핵심 가치 3가지
1. **전환 컨텍스트** — 챗봇 대화 요약을 AI가 즉시 생성, 읽을 필요 없음
2. **감정 온도** — 분노/불만/불안/냉정 4단계, 응대 전략 즉시 판단
3. **추천 오프닝** — 3가지 스타일 오프닝 후보, 클릭 한 번으로 입력창에 삽입

## 랜딩 페이지 구성안

### 파일 위치
- 새 파일: `C:\Users\2rmaq\projects\glance\landing.html`
- 스타일: 인라인 `<style>` 또는 `landing.css` (별도 파일)
- 기존 데모 앱과 같은 폴더에 위치

### 섹션 구조 (순서대로)

#### 1. Hero
- 상단 Nav: 로고(⚡ Glance) + CTA 버튼 "무료로 시작하기"
- 헤드라인: **"상담 전환 순간,\n매니저는 이미 알고 있어야 한다"**
- 서브: "챗봇이 넘긴 고객의 감정·상황·오프닝을 클릭 전에 준비해드립니다"
- CTA 버튼: "데모 체험하기" (→ `index.html`로 링크)
- 우측 또는 하단: 앱 스크린샷 목업 (실제 스크린샷 or CSS 목업)

#### 2. Problem (공감 섹션)
제목: "채널톡 매니저라면 이 장면, 알 거예요"
3개 카드 형태:
- 😤 "챗봇 대화 처음부터 읽다가 고객은 이미 화가 났다"
- 😰 "감정 파악도 못 하고 '어떻게 도와드릴까요?' 시작"
- ⏰ "재인입 3번째, 전 담당자가 뭘 했는지 아무도 모른다"

#### 3. Solution (핵심 기능 3종)
제목: "Glance는 전환 순간을 바꿉니다"
3개 피처 블록 (아이콘 + 제목 + 설명):
- ⚡ **전환 컨텍스트** — AI가 챗봇 대화를 2~3문장으로 요약. 더 이상 처음부터 읽지 않아도 됩니다.
- 🌡️ **감정 온도** — 분노/불만/불안/냉정 4단계로 고객 감정을 즉시 파악. 응대 전략을 먼저 결정하세요.
- ✍️ **추천 오프닝** — 상황에 맞는 오프닝 3가지 제안. 클릭 한 번으로 입력창에 바로 삽입됩니다.

#### 4. How it Works (3단계 흐름)
제목: "클릭 한 번, 준비 완료"
1. 챗봇이 해결 못한 고객이 매니저에게 전환됩니다
2. Glance가 대화 내용을 분석해 컨텍스트를 즉시 표시합니다
3. 감정을 파악하고, 오프닝을 선택해 대화를 시작하세요
